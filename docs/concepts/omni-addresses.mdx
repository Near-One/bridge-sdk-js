# OmniAddress Format

The Omni Bridge uses a unified address format called **OmniAddress** to represent addresses across all supported chains. This makes it unambiguous which chain an address belongs to.

## Format

An OmniAddress is a chain prefix followed by the native address:

```
<chain>:<address>
```

## Examples

| Chain | OmniAddress | Description |
|-------|-------------|-------------|
| Ethereum | `eth:0x1234...abcd` | Ethereum mainnet address |
| Base | `base:0x1234...abcd` | Base L2 address |
| Arbitrum | `arb:0x1234...abcd` | Arbitrum One address |
| Polygon | `pol:0x1234...abcd` | Polygon PoS address |
| BNB Chain | `bnb:0x1234...abcd` | BNB Smart Chain address |
| NEAR | `near:alice.near` | NEAR account |
| Solana | `sol:ABC123...xyz` | Solana public key |
| Bitcoin | `btc:bc1q...` | Bitcoin address |
| Zcash | `zec:t1...` | Zcash transparent address |

## Working with OmniAddresses

The SDK provides utility functions for working with OmniAddresses:

```typescript
import { 
  getChain, 
  getAddress, 
  omniAddress,
  ChainKind 
} from "@omni-bridge/core"

// Parse an OmniAddress
const addr = "eth:0x1234567890123456789012345678901234567890"
const chain = getChain(addr)     // ChainKind.Eth
const raw = getAddress(addr)     // "0x1234567890123456789012345678901234567890"

// Create an OmniAddress
const omni = omniAddress(ChainKind.Eth, "0x1234...")  // "eth:0x1234..."
```

## Token Addresses

Tokens are also represented as OmniAddresses:

```typescript
// USDC on Ethereum
const usdc = "eth:0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48"

// wNEAR on NEAR
const wnear = "near:wrap.near"

// Native SOL (represented as default pubkey)
const sol = "sol:11111111111111111111111111111111"
```

## Native Tokens

For native tokens (ETH, NEAR, SOL), use the zero address or default representation:

| Chain | Native Token Address |
|-------|---------------------|
| EVM chains | `eth:0x0000000000000000000000000000000000000000` |
| Solana | `sol:11111111111111111111111111111111` (default pubkey) |

<Note>
  When bridging native tokens on EVM chains, the transaction's `value` field includes both the transfer amount and any native fee.
</Note>

## ChainKind Enum

The SDK uses a `ChainKind` enum for programmatic chain identification:

```typescript
enum ChainKind {
  Eth = 0,
  Near = 1,
  Sol = 2,
  Arb = 3,
  Base = 4,
  Bnb = 5,
  Btc = 6,
  Zcash = 7,
  Pol = 8
}
```

## Checking Chain Types

```typescript
import { isEvmChain, ChainKind } from "@omni-bridge/core"

isEvmChain(ChainKind.Eth)   // true
isEvmChain(ChainKind.Base)  // true
isEvmChain(ChainKind.Near)  // false
isEvmChain(ChainKind.Sol)   // false
```

## Address Validation

The SDK validates addresses during `validateTransfer()`:

```typescript
const validated = await bridge.validateTransfer({
  token: "eth:0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",
  sender: "eth:0xInvalidAddress",  // ‚ùå Throws ValidationError
  recipient: "near:alice.near",
  // ...
})
```

<Warning>
  Invalid addresses throw a `ValidationError` with code `INVALID_ADDRESS`. Always handle validation errors in your application.
</Warning>
